# Contributing

## Quick Start
1. Fork this repo
2. Create a feature branch: `feat/your-topic`
3. Keep changes small and focused
4. Update README/examples if behavior changes
5. Open a PR with clear before/after notes

## Code Style
- Prefer simple, readable Python
- Add docstrings for non-trivial logic
- Keep CLI interfaces backward-compatible when possible

## PR Checklist
- [ ] Code runs locally
- [ ] README/docs updated
- [ ] No secrets/tokens committed
